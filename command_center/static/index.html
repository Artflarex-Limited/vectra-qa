🔐 Best Practices — Test Accounts

Always create dedicated test accounts with fake data rather than using real user credentials in automated tests. This protects real user data and ensures tests remain reproducible regardless of production state changes.

Use deterministic credentials like test@example.com / password123 that are clearly identifiable as test data, and document them in your team's runbook so anyone can replicate test scenarios without guessing.

Rotate test passwords regularly and store them in a secrets manager rather than hard-coding them in test scripts; Vectra's credential handler never writes credentials to the vault or local storage, keeping them in-memory only.
